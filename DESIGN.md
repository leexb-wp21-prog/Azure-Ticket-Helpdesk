# QuickAid Design Notes

## Visual Direction
Restrained product UI with warm institutional neutrals, one blue service accent, and semantic state colors for priority and status. The product should feel like an internal university system: quiet, reliable, and easy to scan.

## Typography
Use the existing sans-serif stack. Avoid oversized hero treatment in dashboards and forms. Labels, table headers, and metadata should be compact and consistent.

## Components
- Sidebar navigation, top actions, filters, status tabs, ticket rows, and admin tables should share the same radius, border, and hover vocabulary.
- Primary actions use the blue accent. Secondary actions use neutral borders and soft hover surfaces.
- Ticket status and priority indicators should be readable but not decorative.

## Copy
Use service desk terms: request, requester, queue, assigned team, department, status, SLA, update. Avoid words like smart, AI, demo, testing, preview, and backend note in user-facing surfaces.
